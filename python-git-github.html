Configuración del Entorno y Flujo de Trabajo

En esta guía se detallan los pasos necesarios para configurar su entorno de desarrollo, gestionar repositorios con Git y ejecutar scripts de Python correctamente.

Objetivo Principal

Establecer una base sólida para el desarrollo de software versionado.

Metodología

Siga cada paso secuencialmente y marque su avance en el panel lateral.

1. Instalación de Herramientas (recordatorio)

Es fundamental preparar el sistema antes de comenzar la programación.

Python

El lenguaje base de nuestros proyectos.

Descargar Python ↗

Git

Sistema de control de versiones. Incluirá **Git Bash**, una terminal de utilidad para usuarios de Windows.

Descargar Git ↗ (Recomendado para Windows)

IDE (Entorno de Desarrollo Integrado)

Herramienta donde escribirá su código. Se recomiendan:

PyCharm
VS Code

Puede usar otro sin problema alguno.

2. Configuración de GitHub

Plataforma donde se aloja el código versionado de manera remota.

Crear Cuenta

Regístrese para poder subir y compartir sus repositorios.

Ir a github.com ↗

Configurar identidad local

Ejecute estos comandos en la terminal (Git Bash o CMD) solo en la primera ocasión para registrar sus aportes:

git config --global user.name "Su Nombre y Apellido"
git config --global user.email "su-correo@ejemplo.com"

3. Forma de Trabajo (Paso a Paso)

Flujo organizado para trabajar en sus proyectos.

☁️ 3.1. Obtener
💻 3.2. Desarrollo
▶️ 3.3. Ejecución
🔄 3.4. Sincronizar

3.1. Obtener el Proyecto (Fork y Clone)

Fork

Diríjase al repositorio en GitHub y haga clic en Fork. Esto creará una copia en su cuenta.

Clone

Descargue su copia a la computadora escribiendo en la terminal:

git clone https://github.com/su-usuario-github/nombre-del-repositorio.git

3.2. Desarrollo en el IDE

  1. Abra su IDE (VS Code, PyCharm, etc.).
  2. Seleccione la opción "Abrir carpeta" (Open Folder) y localice la carpeta que acaba de clonar.
  3. Modifique o cree los archivos .py necesarios para su labor.

3.3. Ejecución de Scripts

Para verificar que su código funciona correctamente, utilice la terminal integrada o el CMD:

  1. Navegue hasta la carpeta del proyecto.
  2. Ejecute el archivo con el comando:
python nombre_archivo.py

3.4. Sincronización de Cambios

Una vez comprobados los cambios localmente, proceda a subirlos a la nube en este orden:

1. Indexar (Add)

git add .

2. Punto de control (Commit)

git commit -a -m "Descripción breve y clara de la mejora realizada"

3. Subir (Push)

git push
